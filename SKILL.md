# 文档归类整理工具

将 work 文件夹中的文档按规则分类整理到 raw 文件夹，支持增量同步和客户映射表管理。

## 使用方式

```bash
# 默认增量模式
python scripts/organize.py

# 强制全量刷新
python scripts/organize.py --full

# 仅更新客户映射表
python scripts/organize.py --mapping-only

# 预览模式（不执行，只显示将要进行的操作）
python scripts/organize.py --dry-run
```

## 源→目标映射

| 源路径 | 目标路径 |
|--------|----------|
| raw/产品手册/* | 客户档案/产品功能/ |
| raw/产品方案/* | 客户档案/通用业务方案/ |
| raw/客户知识库/{客户}/蓝图/* | 客户档案/优秀客户方案/{客户}/蓝图/ |
| raw/客户知识库/{客户}/蓝图/* | 客户档案/{客户}/蓝图方案/ |
| raw/客户知识库/{客户}/运维工单/* | 客户档案/{客户}/运维工单/ |
| raw/商务信息档案/客户主数据* | 客户档案/{客户}/基础数据/ |
| raw/商务信息档案/订阅台账* | 客户档案/{客户}/订阅合同行/ |
| raw/商务信息档案/固定金额台账* | 客户档案/{客户}/实施合同行/ |
| raw/商务信息档案/人天框架台账* | 客户档案/{客户}/实施合同行/ |
| raw/商务信息档案/项目收款进度查询* | 客户档案/{客户}/订阅合同收款情况/ |
| raw/运维工单/*.xlsx | 客户档案/{识别出的客户}/运维工单/ |

## 客户映射

- 映射表保存在 `raw/_mapping.json`
- 基于客户主数据（商务信息档案/客户主数据）生成
- 运维工单匹配时，只归类在映射表中的客户
- 不在映射表中的客户工单忽略

## 增量同步

- 状态记录在 `raw/_sync_state.json`
- 记录每个文件的 hash 和时间戳
- 只处理新增或修改的文件

## 数据完整性

处理 Excel 时：
- 读取所有 sheet，不遗漏
- 保留原始列名和数据
- 拆分时按客户分组，每个客户一个文件
- 合并时汇总所有数据，不丢失任何行

## 同步模式说明

| 数据类型 | 模式 | 说明 |
|---------|------|------|
| 产品/方案文档 | 增量 | hash 判断，覆盖同名文件 |
| 客户知识库文档 | 增量 | hash 判断 |
| 商务信息档案（主数据） | 快照 | 每次全量覆盖，不保留历史 |
| 运维工单 | 增量+合并 | 按编号去重后合并 |
